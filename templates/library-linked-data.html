Linked Data in Libraries

Linked Data is about representing information in statements called RDF Triples i.e. subject-predicate-object. Libraries standards are shifting to Linked-Data vocabularies like BIBFRAME and Schema.org. RDA and MARC can also be reformulated as Linked Data triples as well.

RDF Triple <subject> <predicate> <object>

Linked data is a general term for connecting one or more publishers of RDF (Resource Description Framework) statements called triples that express a relationship between two resources. and are in the following format:

IRI

International Resource Identifier identifies a resource with URLs being the most common form of an IRI.

What is a Subject?

A subject resource can be:
  • IRI
  • Blank Node
These are all examples of subjects:

http://id.loc.gov/authorities/names/n79032879

http://hds.harvard.edu/

http://id.loc.gov/authorities/names/n79003285

What is a Predicate?

A relationship between the <subject> and <object> is called a property. A predicate must be an IRI.

http://www.w3.org/2000/01/rdf-schema#label

What is a Object?

An object resource can be
  • IRI
  • Blank Node
  • Literal

Together, one or more triples make a Graph.
